## 使用方法

### 快速开始（推荐）

```bash
# 1. 克隆项目
git clone https://github.com/chenming0v0/nai-image.git
cd nai-image

# 2. 配置环境变量
cp .env.example .env
nano .env  # 填写 UPSTREAM_BASE_URL 和 UPSTREAM_API_KEY

# 3. 启动服务
bash start.sh  # Linux/Mac
# 或
start.bat      # Windows

# 4. 访问
# 前端：http://localhost:8080
# 后端：http://localhost:8787
```

### 本地开发

**后端**：
```bash
cd backend
go run ./cmd/server
```

**前端**：
```bash
npm install
npm run dev
```

## 关键配置说明

### 必须配置的环境变量

在 `.env` 文件中：

```env
# 公益站 API 地址（必填）
UPSTREAM_BASE_URL=https://你的公益站地址/v1

# API 密钥（必填）
UPSTREAM_API_KEY=你的密钥

# 默认模型（可选）
DEFAULT_MODEL=nai-diffusion-4-5-full
```

### 端口配置

- 前端：`FRONTEND_PORT=8080`（默认）
- 后端：`BACKEND_PORT=8787`（默认）

如果端口冲突，可以修改这些值。

### 数据持久化

- SQLite 数据库：`./backend/data/nai.db`
- 前端 IndexedDB：浏览器本地存储

## 注意事项

### 1. 用户的重要提醒

根据你的反馈，我需要提醒：

- ✅ **模型在前端选择**：不在环境变量中硬编码，`DEFAULT_MODEL` 只是初始默认值
- ✅ **API 配置存在 SQLite**：不使用环境变量存储敏感信息
- ✅ **超时配置在 SQLite**：可在前端设置页面配置
- ✅ **只有端口写在环境变量**：`FRONTEND_PORT` 和 `BACKEND_PORT`

当前配置已经符合这些要求：
- 环境变量只提供**初始默认值**
- 真正的配置存储在后端 SQLite 数据库
- 前端可以通过 API 动态获取和修改配置

### 2. Docker 部署优势

- **一键启动**：`docker-compose up -d`
- **数据持久化**：SQLite 挂载到宿主机
- **隔离环境**：不污染系统环境
- **易于维护**：统一的部署方式

### 3. 安全建议

- 不要将 `.env` 文件提交到 Git
- 定期备份 `backend/data/nai.db`
- 生产环境建议使用 HTTPS
- 限制端口访问（防火墙/反向代理）
