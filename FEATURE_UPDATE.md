# 功能更新：图片上传智能选择 + 标签显示修复

## 更新内容

在 Gallery 模式下实现了两个重要更新：
1. **上传第二张图片时的智能询问**：避免用户误操作混用参考图和 Vibe 模式
2. **图片标签显示修复**：正确显示图片的实际用途

## 实现逻辑

### 背景
在 NAI Gallery 模式中，图片的用途取决于图片的**总数量**：
- **只有 1 张图片**：作为参考图（改图底图），用于改图或添加遮罩
- **有 2 张及以上图片**：全部作为 Vibe 参考，用于风格参考

**重要约束**：参考图和 Vibe 只能选择一个模式，不能混用。

## 主要修改

### 1. 智能询问对话框 (handleFiles 函数)

当检测到以下条件时触发询问对话框：
1. 当前处于 Gallery 模式
2. 已经上传了 1 张图片（参考图）
3. 用户尝试上传第二张图片

#### 对话框选项

**选项 1：替换参考图（次要操作）**
- **操作**：用新图片替换掉第一张参考图
- **结果**：保持参考图模式，新图片成为唯一的参考图
- **适用场景**：用户发现第一张图片选错了，想要换一张参考图

**选项 2：添加为 Vibe（主要操作）**
- **操作**：将新图片添加到图片列表
- **结果**：现在有两张图片，都作为 Vibe 参考使用
- **适用场景**：用户想要使用多张图片作为风格参考
- **注意**：这种模式下，第一张图片也会变成 Vibe 参考，不再是改图底图

### 2. 图片标签显示修复 (renderImageThumb 函数)

**修复前的错误显示**：
- 第 1 张图：显示"改图"
- 第 2 张图：显示"参考1"
- 第 3 张图：显示"参考2"
- ❌ 问题：误导用户以为第一张是改图，其他是 Vibe

**修复后的正确显示**：
- **只有 1 张图时**：
  - 第 1 张：显示"改图"（蓝色标签）
  - 如果有遮罩：显示"改图·局部"（紫色标签）

- **有 2 张及以上图时**：
  - 第 1 张：显示"Vibe 1"（琥珀色标签）
  - 第 2 张：显示"Vibe 2"（琥珀色标签）
  - 第 3 张：显示"Vibe 3"（琥珀色标签）
  - ✅ 正确：清晰表明全部都是 Vibe 参考

**关键代码逻辑**：
```typescript
// 判断是否为改图底图：只有 1 张图 且 是第一张
const isGalleryBase = isGalleryMode && inputImages.length === 1 && idx === 0

// 判断是否为 Vibe 参考：有 2 张及以上图
const isGalleryVibeRef = isGalleryMode && inputImages.length >= 2
```

## 代码修改位置

**文件**：`src/components/InputBar.tsx`

### 修改 1：智能询问对话框
**函数**：`handleFiles` (第 1102-1169 行)

**改动内容**：
- 添加了 Gallery 模式下上传第二张图片时的判断逻辑
- 使用 `createInputImageFromFile` 预处理图片
- 调用 `setConfirmDialog` 显示选择对话框
- 提供"替换参考图"和"添加为 Vibe"两个选项

### 修改 2：图片标签显示逻辑
**函数**：`renderImageThumb` (第 1690-1931 行)

**改动内容**：

1. **判断逻辑修复**（第 1690-1701 行）：
```typescript
// 修改前：
const isGalleryBase = isGalleryMode && idx === 0
const isGalleryVibeRef = isGalleryMode && idx >= 1

// 修改后：
const isGalleryBase = isGalleryMode && inputImages.length === 1 && idx === 0
const isGalleryVibeRef = isGalleryMode && inputImages.length >= 2
```

2. **标签渲染修复**（第 1883-1897 行）：
```typescript
// 修改前：
{isGalleryBase && (
  <span>{isMaskTarget ? '改图·局部' : '改图'}</span>
)}
{isGalleryVibeRef && (
  <span>参考{idx}</span>
)}

// 修改后：
{isGalleryBase && !isMaskTarget && (
  <span>改图</span>
)}
{isGalleryBase && isMaskTarget && (
  <span>改图·局部</span>
)}
{isGalleryVibeRef && (
  <span>Vibe {idx + 1}</span>
)}
```

**关键改进**：
- 标签文字从"参考{idx}"改为"Vibe {idx + 1}"，更直观
- 区分了有无遮罩的改图底图显示

## 用户体验提升

1. **避免误操作**：防止用户不小心混用参考图和 Vibe 模式
2. **清晰提示**：对话框中明确说明两种模式的区别和限制
3. **灵活选择**：用户可以根据实际需求选择合适的模式
4. **即时反馈**：每次操作后都有 toast 提示，让用户知道操作结果
5. **标签正确显示**：图片标签准确反映实际用途，不再误导用户

## 视觉效果说明

### 标签颜色含义
- **蓝色标签（改图）**：表示这是改图底图，用于 img2img
- **紫色标签（改图·局部）**：表示这是带遮罩的改图底图
- **琥珀色标签（Vibe 1/2/3...）**：表示这是 Vibe 风格参考

### 对话框外观
- 标题：**选择图片用途**
- 内容：清晰说明参考图和 Vibe 的互斥关系
- 按钮：
  - 左侧（次要）："替换参考图"（灰色调）
  - 右侧（主要）："添加为 Vibe"（蓝色调，推荐）

## 注意事项

- 此功能仅在 Gallery 模式下生效
- 仅在第一张图片存在且上传第二张图片时触发
- 对话框的设计遵循项目现有的 UI 风格
- 使用了项目已有的 `setConfirmDialog` 和 `showToast` 机制
