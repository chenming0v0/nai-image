## 🚀 使用方法

### 一键启动（推荐）

```bash
# 1. 克隆项目（如果还没有）
git clone https://github.com/chenming0v0/nai-image.git
cd nai-image

# 2. 安装依赖（仅第一次）
npm install

# 3. 启动开发环境
npm start
```

启动后：
- **前端**: http://localhost:5173
- **后端**: http://localhost:8787
- **健康检查**: http://localhost:8787/api/health

### 分别启动

如果需要分别调试：

```bash
# 终端 1: 启动后端
npm run dev:backend

# 终端 2: 启动前端
npm run dev:frontend
```

### 停止服务

按 `Ctrl+C` 即可同时停止前后端服务。

## 📋 所有可用命令

### 开发命令

| 命令 | 说明 |
|------|------|
| `npm start` | 🌟 一键启动前端 + 后端 |
| `npm run dev:all` | 同上 |
| `npm run dev:frontend` | 只启动前端 |
| `npm run dev:backend` | 只启动后端 |

| 命令 | 说明 |
|------|------|
| `npm run build:all` | 构建前端 + 后端 |
| `npm run build` | 只构建前端 → `dist/` |
| `npm run build:backend` | 只构建后端 → `dist/nai-backend` |

| 命令 | 说明 |

| `npm run preview` | 预览生产构建 |

## 🔧 环境配置（可选）

### 后端配置

如果需要配置上游 API：

**创建 `backend/.env` 文件**：
```env
UPSTREAM_BASE_URL=https://api.example.com/v1
UPSTREAM_API_KEY=your-key
DEFAULT_MODEL=nai-diffusion-4-5-full
PORT=8787
```

**或使用环境变量**：
```bash
# Windows PowerShell
$env:UPSTREAM_BASE_URL="https://api.example.com/v1"; npm start

# Linux/Mac
UPSTREAM_BASE_URL=https://api.example.com/v1 npm start
```

### 前端配置

创建 `.env.local` 文件：
```env
VITE_API_BASE_URL=http://localhost:8787
```

## ❓ 常见问题

### Q: 端口被占用怎么办？

**前端端口冲突**（5173）：

修改 `vite.config.ts`：
```ts
export default defineConfig({
  server: {
    port: 5174  // 改成其他端口
  }
})
```

**后端端口冲突**（8787）：

设置环境变量：
```bash
# Windows
$env:PORT=8788; npm start

# Linux/Mac
PORT=8788 npm start
```

### Q: Go 命令找不到？

确认已安装 Go：
```bash
go version
```

如果未安装，访问 https://go.dev/dl/ 下载安装。

### Q: concurrently 安装失败？

```bash
# 清理并重新安装
rm -rf node_modules package-lock.json
npm install
```

### Q: 后端启动失败？

```bash
# 检查 Go 依赖
cd backend
go mod download
go run ./cmd/server
```
